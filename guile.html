1.1.2 脚本

  1. 基本格式

    脚本的基本格式如下: 

           #!/usr/bin/guile -s
       !#
                                               ; coding:utf-8
       (display "Hello, world!")
       (newline)
    1. 脚本的第一行得是 #!/usr/bin/guile -s 或类似的东西
    2. 脚本中程序正式开始前要有单独的一行 !# 这是为了能在这里进行换行 \
    3. 非 posix 需要在前5行中出现 coding:utf-8
  2. 命令行参数处理
    1. (command-line) 获取所有命令行参数
              #! /usr/local/bin/guile -s
        !#
        (write (command-line))
        (newline)

      (command-line) 会返回所有命令行参数的列表

    2. 使用主函数方式
              #! /usr/local/bin/guile \
        -e main -s
        !#
        (define (main args)
          (write args)
          (newline))

      通过 -e 参数,指定了主函数, 命令行参数会作为主函数的参数进行调用

    3. (ice-9 getopt-long) 模块

      见下文

1.1.3 在 c 中使用 Guile 作为扩展

样例 c 程序 

      #include <stdlib.h>
      #include <libguile.h>
      static SCM my_hostname (void)
      {
        char *s = getenv ("HOSTNAME");
        if (s == NULL)
          return SCM_BOOL_F;
        else
          return scm_from_locale_string (s);
      }
      static void inner_main (void *data, int argc, char **argv)
      {
        scm_c_define_gsubr ("my-hostname", 0, 0, 0, my_hostname);
        scm_shell (argc, argv);
      }
      int main (int argc, char **argv)
      {
        scm_boot_guile (argc, argv, inner_main, 0);
        return 0; /* never reached */
      }

使用 gcc 编译的方法为 gcc -o simple-guile simple-guile.c `pkg-config --cflags --libs guile-2.2`

1.1.4 用 c 为 Guile 编写扩展

样例程序 

      #include <math.h>
      #include <libguile.h>
      SCM j0_wrapper (SCM x)
      {
        return scm_from_double (j0 (scm_to_double (x)));
      }
      void init_bessel ()
      {
        scm_c_define_gsubr ("j0", 1, 0, 0, j0_wrapper);
      }

编译方法: gcc `pkg-config --cflags guile-2.2` \ -shared -o libguile-bessel.so -fPIC bessel.c

编译后的 so 文件可以直接在 guile repl中调用 

      (load-extension "./libguile-bessel" "init_bessel")
      (j0 2)

1.2.3 宏

  1. 语法宏

    在 Scheme中定义语法宏非常容易 

           (define-syntax nil! ;宏名
         (syntax-rules (doit) ;规则,第一个参数为保留关键字,可以进行匹配
           ((_ x) ;变换前的形式,对输入的内容进行模式匹配,下划线表示宏名
            (set! x '())) ;变换后的形式
           ((_ ...) (...)) ;可以匹配多种形式,三个点表示剩下的所有东西
           ((_ x do y) (x y)) ;可以匹配关键字
           ))
    语法元素 解释
    define-syntax 表示开始定义宏
    syntax-rules 表示语法匹配,第一个参数为关键字,从第二个开始的括号,里面分别对应变换前和变换后的形式
    _ 下划线表示宏名
    ... 三个点表示剩下的所有东西
    define-syntax-rule 定义简短的宏
    syntax-error 匹配成功时可以返回错误信息
    syntax-case 可以进行匹配检查

1.2.4 call/cc 继续

call/cc 会将现在运行到的东西返回到顶层,它要接受一个匿名函数作为参数 先不管这是什么鬼,高大上的 call/cc 其实就是 return 没有触发 call/cc 时: 

      (= (* 3 (+ 1 4))
         (* 3 (call/cc (lambda (return)
                         (+ 1 4)))))

一旦触发了,就不按求值顺序返回了,直接把指定的这个东西返回回去: 

      (= (* 3 2)
         (* 3 (call/cc (lambda (return) 
                         (+ 1 (return 2))))))

  1. 正则表达式字符串没有字面量，只是普通字符串，因此得用很多转移字符才行

  3. 转义一般得用两个斜线 //., //(

              (define date-regex
          "([0-9][0-9][0-9][0-9])([0-9][0-9])([0-9][0-9])")

1.5.3 (ice-9 getopt-long) 命令行参数解析

命令行参数解析只有两个函数, getopt-long 和 option-ref 

      #!/usr/bin/guile \
      -e main -s
      !#
      (use-modules (ice-9 getopt-long))
      (define (main arg)
        (let* ((isstring? (lambda (s) (string? s)))
               (option-spec
                `((version (single-char #\v) (value #f))
                  (help    (single-char #\h) (value #f))
                  (file-name (value #t)
                             (single-char #\n)
                             (required? #f)
                             (predicate ,isstring?))))
               (options (getopt-long arg option-spec #:stop-at-first-non-option #t))
               (help-wanted (option-ref options 'help #f))
               (version-wanted (option-ref options 'version #f))
               (file-name (option-ref options 'file-name "nofile")))

          (when version-wanted
            (display "v1.0.0\n")
            (quit))
          (when help-wanted
            (display
             (string-join '("getopt-long-example [options]"
                            "-v,  --version  Display version"
                            "-h,  --help     Display this help"
                            "")
                          "\n"))
            (quit))
          (when file-name
            (display file-name))))
  1. getopt-long

    定义参数格式为 '((option-name (key value) ...))

    参数 作用 注意事项
    (single-char char) 指定短名称 -h  
    (required? bool) 如果设为必需,则没有时会报错  
    (value bool) 这个参数是否带值  
    (predicate func) 如果参数带值,就会调用这个函数,并把值做为参数  
    #:stop-at-first-non-option #t 解析读取到第一个为真的值 用于 -h,-v 等同时出现时
  2. option-ref

    用来得到值,或是否输入选项
