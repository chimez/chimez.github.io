1 NextCloud 配置记录

1.1 LAMP 安装与配置

参考 Debian/LaMp

1.1.1 MariaDB

apt install mariadb-server mariadb-client

1.1.2 apache2

apt install apache2 apache2-doc
  1. Configuring user directories for Apache Web Server

    Enable module 

    a2enmod userdir

    Configure Apache module userdir in /etc/apache2/mods-enabled/userdir.conf as follows: 

    <IfModule mod_userdir.c>
	UserDir public_html
	UserDir disabled root

	<Directory /home/*/public_html>
		AllowOverride All
		Options MultiViews Indexes SymLinksIfOwnerMatch
		<Limit GET POST OPTIONS>
			Require all granted
		</Limit>
		<LimitExcept GET POST OPTIONS>
			Require all denied
		</LimitExcept>
	</Directory>
</IfModule>

    Create directory as user (not as root): 

    mkdir /home/$USER/public_html

    Change group as root (substitute your username) and restart web server: 

    chgrp www-data /home/<username>/public_html
service apache2 restart

1.1.3 php

apt install php php-mysql
  1. test php

    edit /var/www/html/test.php as: 

    <?php phpinfo(); ?>

    visit http://localhost/test.php

1.2 nextcloud

1.2.1 安装 php 模块

apt install php-curl php-xml php-gd php-json php-mbstring php-zip php-mysql php-bz2 php-intl php-ldap php-smbclient php-imap php-bcmath php-gmp php-imagick

1.2.2 设置数据库

一般默认无密码，首次启动可以用 mysql_secure_installation 设置密码。

登录数据库 

mysql -uroot -p

创建账号 

CREATE USER 'nextcloud'@'localhost' IDENTIFIED BY 'password';
CREATE DATABASE IF NOT EXISTS nextcloud CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
GRANT ALL PRIVILEGES ON nextcloud.* TO 'nextcloud'@'localhost';
FLUSH PRIVILEGES;

1.2.3 安装 nextcloud

下载 the Nextcloud .tar archive ,解压到 nextcloud 别忘了测试 MD5 

md5sum -c nextcloud-x.y.z.tar.bz2.md5 < nextcloud-x.y.z.tar.bz2

把文件都复制到 /var/www 并设置权限 

cp -r nextcloud /var/www
chown -R www-data:www-data /var/www/nextcloud/

1.2.4 配置 apache2

新建文件 /etc/apache2/sites-available/nextcloud.conf 

Alias /nextcloud "/var/www/nextcloud/"

<Directory /var/www/nextcloud/>
  Require all granted
  AllowOverride All
  Options FollowSymLinks MultiViews

  <IfModule mod_dav.c>
    Dav off
  </IfModule>

</Directory>

启用配置 

a2ensite nextcloud.conf

1.2.5 开启 apache2 模块

a2enmod rewrite
a2enmod headers
a2enmod env
a2enmod dir
a2enmod mime

1.2.6 重启服务

service apache2 restart
service mysql restart

1.2.7 测试

打开 http://localhost/nextcloud

2 NextCloud 配置记录 - Docker 版本

2.1 安装

  1. 选择在docker中安装它: https://github.com/nextcloud/docker
    1. 首先需要安装 docker-ce, 按照官网指南即可: https://docs.docker.com/install/linux/docker-ce/debian/
    2. 然后安装 Docker Compose, https://docs.docker.com/compose/install/
  2. 创建好 docker-compose.yml 文件并运行, 参考 Running this image with docker-compose
    1. 由于墙的问题, 先利用 docker pull 把包分别下好
  3. 运行 docker-compose up -d
  4. 在docker里执行命令用 docker-compose exec app ls

2.2 挂载本地文件的方法

  1. 先在yml里指定好文件位置, 规则为 - 本地路径:docker路径
  2. 进入 docker shell 的方法 docker exec -it DOCKER /bin/bash
  3. 进入docker里设置权限 chown -R www-data:www-data docker路径
  4. 强制进行扫描 sudo -u www-data php console.php files:scan --all, 没有 sudo 就 apt 安装一个

2.3 备份

需要备份的有

  1. config
  2. data
  3. theme
  4. mysql

使用这个脚本即可 

#!/bin/bash
DATE=`date +"%Y%m%d"`
cd /root/docker/backup
mkdir $DATE
cd $DATE
docker exec nextcloud_db_1 mysqldump -hdb -uroot -plongman nextcloud > "nextcloud-sqlbkp_"$DATE".bak"
cd /root/docker/nextcloud-data
tar -cvJf "nextcloud_data_"$DATE".tar.xz" var/www/html/data var/www/html/config
mv "/root/docker/nextcloud-data/nextcloud_data_"$DATE".tar.xz" "/root/docker/backup/"$DATE"/"
cd /root/docker/backup

再拷贝到什么地方去 

rsync -av --progress /root/docker/backup/ /target/backup

2.4 升级到新版本

docker-compose build --pull
docker-compose up -d

3 问题与解决方案

3.1 上传大文件 >500M

  1. 参见网址 ht
  2. 首先在网页设置里修改文件大小
  3. 然后修改 php.ini 里的大小
  4. 再修改 nginx 的配置的大小

3.2 使用本地FTP

  1. 安装 vsftpd
  2. 配置 /etc/vsftpd.conf 启用 utf8
  3. 从docker中访问宿主机的地址为 127.17.0.1
  4. 或者是 ifconfig docker0 中 inet 后的ip地址

3.3 Nginx 反向代理与自签名 https 证书

3.3.1 nginx 基本使用

  1. 安装 nginx: apt 解决
  2. 配置文件在 /etc/nginx/nginx.conf 里, 或者 nginx -V 查看
  3. 以指定的配置文件运行 nginx -c /to/nginx.conf

3.3.2 nginx 反向代理配置

在 nginx.conf 文件里写入 

http{
  include /a/a.conf;
}

在 /a/a.conf 里写入下面的内容可以实现http的 80 端口向 8080 转发, 并能记录日志到文件 

server {
    listen 80;
    server_name  localhost;

    access_log  /root/logs/quancha.access.log;

    ## send request back to apache ##
    location / {
	proxy_pass  http://localhost:8080;
   }
}

3.3.3 https 转发反向代理

与普通的http相同只是增加了 ssl 设置字段 

server {
    listen 80;
    server_name  localhost;

    access_log  /root/nginx/logs/nextcloud.access.log;
    error_log  /root/nginx/logs/nextcloud.error.log;

    ssl on;
    ssl_certificate      /root/nginx/ssl/server.crt;
    ssl_certificate_key  /root/nginx/ssl/server.key;

    ssl_session_timeout  5m;
    ## send request back to apache ##
    location / {
	proxy_pass  http://localhost:8080;
    }
}

3.4 自签名https证书

3.4.1 第1步：生成私钥

openssl genrsa -des3 -out server.key 2048

3.4.2 第2步：生成CSR（证书签名请求）

openssl req -new -key server.key -out server.csr

需要依次输入国家，地区，城市，组织，组织单位，Common Name和Email。其中Common Name，可以写自己的名字或者域名，如果要支持https，Common Name应该与域名保持一致，否则会引起浏览器警告。 

Country Name (2 letter code) [AU]                          :CN
State or Province Name (full name) [Some-State]            :Beijing
Locality Name (eg, city) []                                :Beijing
Organization Name (eg, company) [Internet Widgits Pty Ltd] :joyios
Organizational Unit Name (eg, section) []                  :info technology
Common Name (e.g. server FQDN or YOUR name) []             :demo.joyios.com
Email Address []                                           :liufan@joyios.com

3.4.3 第3步：删除私钥中的密码

在第1步创建私钥的过程中，由于必须要指定一个密码。而这个密码会带来一个副作用，那就是在每次Apache启动Web服务器时，都会要求输入密码，这显然非常不方便。要删除私钥中的密码，操作如下： 

cp server.key server.key.org
openssl rsa -in server.key.org -out server.key

3.4.4 第4步：生成自签名证书

openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt

3.5 apache2 自签名 https 证书

3.5.1 启用 ssl 模块

a2enmod ssl
a2ensite default-ssl.conf

3.5.2 配置证书

  1. 在 /etc/apache2/ 下新建一个 ssl 文件夹，把这两个文件 apache.crt, apache.key 扔进去
  2. 打开 /etc/apache2/sites-available/default-ssl.conf
  3. 在 ServerAdmin 下另起一行加上 ServerName 你的域名:443
  4. 找到 SSLEngine , SSLCertificateFile , SSLCertificateKeyFile 并修改成以下内容
SSLEngine on

SSLCertificateFile /etc/apache2/ssl/apache.crt

SSLCertificateKeyFile /etc/apache2/ssl/apache.key

完成

3.5.3 配置 HTTPS 强制跳转

打开 /etc/apache2/sites-available/000-default.conf ， 在 <VirtualHost *:80></VirtualHost> 标签内随便一个地方加入以下三行 

RewriteEngine on
RewriteCond  %{HTTPS} !=on
RewriteRule  ^(.*) https://%{SERVER_NAME}$1 [L,R]

3.5.4 使配置生效

  1. 启动 rewrite mod : sudo a2enmod rewrite
  2. 重启 apache : sudo etc/init.d/apache2 restart
  3. 配置结束。不出意外配置应该已经生效。
