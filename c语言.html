1.1.3 函数与程序结构

  1. 定义函数
    void fun(int x){
  x;
}
  2. 外部变量
    1. 声明外部变量时,同一文件内的可以直接用
    2. 不同文件的需要用 extern
  3. 头文件
    1. 头文件用 .h, 里面放上所有的声明
  4. 静态 static
    1. 声明的静态变量 static int b=1;, 只能在文件内被使用,不能被其它文件所用
    2. 也可用于声明函数, 同样会使这个函数只能在同一文件内可用
  5. 寄存器变量 register
    1. 寄存器变量是使用频率更高的变量,用来提示编译器进行优化
    2. 编译器可以忽略它
    3. register int x;
    4. 只能用在内部变量以及函数参数上
    5. 可以声明过量的寄存器变量, 反正编译器会自行决定要不要忽略它
    6. 不管是不是真的放在了寄存器里,这个变量的地址都不可访问了
  6. 预处理器
    1. 文件包含 #include
    2. 宏替换 #define
      1. 宏替换只对记号可用, 对字符串不会替换
      2. 语法为 #define forever for (;;)
      3. 宏替换可以带参数 #define max(A,B) ((A)>(B)?(A):(B))
      4. 取消宏替换 #undef max
      5. 宏的形式参数不能带引号, 但是可以用井号作为前缀获得其带引号的名字
      6. #define dprint(expr) printf(#expr " = %g\n", expr)
      7. 两井号可以将参数名连接起来 #define paste(front, back) front ## back,会得到 paste(name, 1)=>name1
    3. 条件包含 if
      1. #if 后可以加上判断式, 非0为真,0为假
      2. #if – #elif – #else – #endif
      3. #if 中可以用 defined(name) 来获得是否已定义
      4. #ifdef name / #ifndef name – #endif 相当于 #if defined(name)

2.2 autotools 基本使用方法

基本的工具是两个项目: autoconf 和 automake

2.2.1 基本使用流程

  1. 执行命令: autoscan
  2. 修改生成的 configure.scan 文件后缀为 .ac: mv configure.scan configure.ac
  3. 修改 configure.ac 文件,加入项目名称等, 以及 AM_INIT_AUTOMAKE
  4. 执行命令: aclocal
  5. 执行命令: autoheader

  6. 新建并修改文件: vi Makefile.am 

    AUTOMAKE_OPTIONS=foreign subdir-objects   //各种选项 gnu 等
    bin_PROGRAMS= hello   //定义要产生的执行文件名。如果要产生多个执行文件，每个文件名用空格隔开
   hello_SOURCES= include/pr.h src/hello.c    //定义“hello”这个执行程序所需要的原始文件。
  7. 执行命令: automake
  8. 执行命令: autoconf
  9. 构建完成,执行命令: ./configure && make

2.2.2 各种问题的解决

  1. 源代码在子文件里, 分割 Makefile

    1. 在 configure.ac 中加入 

      AC_CONFIG_FILES([Makefile src/Makefile])

    2. 在最上层 Makefile.am 中加入 

      SUBDIRS = src
    3. 在 src/Makefile.am 中写编译相关的内容
  2. 使用 pkg-config 加载库

    参见 pkg-config 文档

    1. 在 configure.ac 中加入 

      PKG_CHECK_MODULES([X], [x])
PKG_CHECK_MODULES([GLIB], [glib-2.0])

    2. 在 Makefile.am 中加入 

      myapp_CFLAGS = $(X_CFLAGS)
 myapp_LDADD = $(X_LIBS)
  3. 在 configure.am 中指定 automake 的选项

    1. 在 Makefile.am 中的写法 

      AUTOMAKE_OPTIONS=foreign subdir-objects   //各种选项 gnu 等

    2. 在 configure.am 中的写法 

      AM_INIT_AUTOMAKE([foreign subdir-objects])
  4. debug 与 release 方式

    无需配置,只是编译时的命令

    1. debug 

      mkdir debug 
cd debug 
path/to/configure --prefix=/path/to/debug  CFLAGS="-ggdb3 -O0" && make && make install
      1. –ggdb3 是为 gdb 优化生成, 3为级别
      2. 可以改成 -g 等
    2. release
    mkdir release 
cd release 
path/to/configure --prefix=/path/to/release  CFLAGS="-O3" && make && make install

6.1 flex

6.1.1 独立使用的规则

flex是词法分析器,文档结构为 a.y 

/*  声明段: */
/*  这里放c代码,会不变的拷贝过去,可以放头文件,结构体定义,函数定义等 */
%{
#include <stdio.h>
  %}
/*  标识符和状态声明: */
/*  状态由 %x 开始 */
%x COMMENT
 /*  定义规则名称, 小括号定义规则,大括号使用名称 */
digit		([0-9])
  integer		({digit}+)
 /*  规则定义:           */
  %% 
  [a-z] {printf("aaz\n");}
{integer} {printf("it\n");}
"integer" {printf("as\n");}
"//" {BEGIN COMMENT; printf("cc\n");} 
<COMMENT>\n {BEGIN INITIAL;printf("in\n");}
%%
 /*  c函数定义: 可留空 */
int yywrap(void) {return 1;} /*  全部读取完的行为 */
  1. 注释只能是 /* */
  2. 行首不能有空格
  3. 定义状态 %x COMMENT
  4. 初始状态为 INITIAL
  5. 转移状态用 BEGIN COMMENT
  6. 指定状态的规则用 <COMMENT>[0-9] {return A}
  7. 生成 c 文件 lex a.y
  8. 编译它 gcc lex.yy.c -o outfile -lfl

6.1.2 作为yacc前端的规则

%{
#include <stdio.h>
#include “y.tab.h”
%}

%%
[0-9]+         return NUMBER;
heat           return TOKHEAT;
on|off         return STATE;
target         return TOKTARGET;
temperature    return TOKTEMPERATURE;
\n             /* ignore end of line */;
[ \t]+         /* ignore whitespace */
%%
  1. 引用 y.tab.h 会由 yacc 自动生成
  2. 要传入给 yacc 的东西就 return 个名字
