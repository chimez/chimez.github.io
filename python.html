8 asyncio: 协程

8.1 异步与协程

python 的 asyncio 是基于协程的库，协程就是在同一个线程里进行调度，把一些阻塞的任务后延来提高并发数。 因此这不是多线程也不是多进程，指望它来实现并行计算是不现实的。

8.2 基本使用

8.2.1 直接的使用协程

这里使用 asyncio.sleep(1) 模拟阻塞的进程，定义函数前面加上 async ，要等待的部分加上 await ，运行函数使用 asyncio.run() 就可以了。 这里能看出直接使用协程仍然是按照顺序执行，并没有减少时间，还是运行了6秒。 

import time
import asyncio

async def func(x):
    print(f"{x}:in")
    await asyncio.sleep(x)
    print(f"{x}:out")
    return x+10

async def main():
    result1 = await func(1)
    print("finish 1:",result1)
    result2 = await func(2)
    print("finish 2:",result2)
    result3 = await func(3)
    print("finish 3:",result3)

print(time.strftime("%X"))
asyncio.run(main())
print(time.strftime("%X"))

     00:24:47
     1:in
     1:out
     finish 1: 11
     2:in
     2:out
     finish 2: 12
     3:in
     3:out
     finish 3: 13
     00:24:53

8.2.2 使用 task 同时运行协程

使用 asyncio.create_task(func()) 就创建了 task ，能够看到作为 task 被自动管理之后函数会同时开始，并且会加速一定的时间。 

import time

    print("finish 3:",result3)

8.2.3 使用 futures 和 线程池将同步函数改造成异步函数

上面的 task 就已经运行得到了我们想要的结果，但是很可惜阻塞的内容必须得是可以写在 await 后面才行。 像下面这样，我们把一个同步的函数包装了一下，再用 task 运行，发现函数仍是一个一个执行的， 只是自动调度把它们全都执行完之后再回到主线程接着运行。 

import time
import os
import asyncio

async def f(x):
    os.system(f"sleep {x}")
    return x+10

async def func(x):
    print(f"{x}:in")
    result = await f(x)
    print(f"{x}:out")
    return result

async def main():
    task1 = asyncio.create_task(func(1))
    task2 = asyncio.create_task(func(2))
    task3 = asyncio.create_task(func(3))
    result1 = await task1
    print("finish 1:",result1)
    result2 = await task2
    print("finish 2:",result2)
    result3 = await task3
    print("finish 3:",result3)

print(time.strftime("%X"))
asyncio.run(main())
print(time.strftime("%X"))

00:26:45
1:in
1:out
2:in
2:out
3:in
3:out
finish 1: 11
finish 2: 12
finish 3: 13
00:26:51

这是因为同步函数把线程阻塞了，协程只能在同一个线程里起作用，如果某个协程会阻塞整个线程的话，那就和顺序执行没区别了。

所以解决这个问题只需要开个新线程，把会阻塞线程的东西放到别的线程里执行，这样就改造出了一个很好的异步函数。 下面使用 concurren.futures 中的线程池，这里还有一个进程池，不过进程池使用消息通信，要求参数必须能够序列化，在某些情况下没法用的。 首先使用 asyncio.get_event_loop() 获得当前的事件循环，然后在这个事件循环中使用 run_in_executor 函数注册需要运行的函数和相应的池， 最后使用 asyncio.gather 函数让这些 futures 同时运行起来。 

import time
import os
import asyncio
from concurrent.futures import ThreadPoolExecutor

def func(x):
    print(f"{x}:in")
    os.system(f"sleep {x}")
    print(f"{x}:out")
    return x+10


async def main():
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor() as pool:
        futures = (
            loop.run_in_executor(pool, func, 1),
            loop.run_in_executor(pool, func, 2),
            loop.run_in_executor(pool, func, 3),
        )
        await asyncio.gather(*futures)
        for future in futures:
            print(future.result())
            
print(time.strftime("%X"))
asyncio.run(main())
print(time.strftime("%X"))

00:33:56
1:in
2:in
3:in
1:out
2:out
3:out
11
12
13
00:33:59

当然这样改造之后的函数可以包装成一个普通异步函数，再用 task 来调用。 

import time
import os
import asyncio
from concurrent.futures import ThreadPoolExecutor

def func(x):
    print(f"{x}:in")
    os.system(f"sleep {x}")
    print(f"{x}:out")


async def afunc(x):
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor() as pool:
        futures = (
            loop.run_in_executor(pool, func, x),
        )
        await asyncio.gather(*futures)

async def main():
    task1 = asyncio.create_task(afunc(3))
    task2 = asyncio.create_task(afunc(2))
    task3 = asyncio.create_task(afunc(1))
    await task1
    print("finish 1")
    await task2
    print("finish 2")
    await task3
    print("finish 3")


print(time.strftime("%X"))
asyncio.run(main())
print(time.strftime("%X"))

     3:in
     2:in
     1:in
     1:out
     2:out
     3:out
     finish 1
     finish 2
     finish 3
     01:21:52

9 multiprocessing: 并行

9.1 基本使用

import multiprocessing as mul 
pool =mul.Pool(processes=3) # 创建进程池
def fun(p):                 # 每个子进程要执行的函数
    return p
par = [p1,p2,p3]             # 参数表,每个子进程传入一个参数
results = []                 # 保存结果list
results = pool.map(fun, par) # 快速并行计算,fun为函数,par为传入参数表

pool.join()                  # 在Linux中要将子进程合并进主进程,避免僵尸进程

11 PLY Python 中的 lex/yacc

11.1 lex 词法分析器

11.1.1 用法说明

  1. 由于ply使用了大量的反射,所以写起来的东西除了看起来是python 格式之外,已经是个DSL了
  2. 词法分析有两部分,先在 tokens 里定义所有的名字,再加上 t_ 前缀,逐个定义所有的匹配方法,单纯是字符的,对应于正则表达式,带条件判断的和带返回值的写成函数形式
  3. 还能使用状态来解析大括号或注释

11.1.2 编写参考

  1. tokens 名称列表

    全部要大写 

           tokens = (
           'NUMBER',
           'PLUS',
           'MINUS',
           'TIMES',
           'DIVIDE',
           'LPAREN',
           'RPAREN',
       )
  2. token 规则
    1. 无返回值,无特殊判断 -> 正则表达式
              t_PLUS = r'\+'
    2. 有需要匹配的值 -> 函数
      1. 第一行是正则规则
      2. 第二行是计算规则,注意输入的参量是个奇怪的东西
      3. 最后返回值
              def t_NUMBER(t):
            r'\d+'
            t.value = int(t.value)
            return t
    3. 保留关键字 -> 字典+函数
      1. 注意在匹配保留关键字时,不要写成正则形式,因为正则形式会匹配多余的东西
      2. 例如: r'for' 匹配到了 forget
              reserved = {
            'if' : 'IF',
            'then' : 'THEN',
            'else' : 'ELSE',
            'while' : 'WHILE',
            ...
        }

        tokens = ['LPAREN','RPAREN',...,'ID'] + list(reserved.values())

        def t_ID(t):
            r'[a-zA-Z_][a-zA-Z_0-9]*'
            t.type = reserved.get(t.value,'ID')    # Check for reserved words
            return t
    4. 匹配忽略 -> pass 函数或 t_ignore
              def t_COMMENT(t):
            r'\#.*'
            pass

        t_ignore_COMMENT = r'\#.*'
    5. 行或列位置
              # Define a rule so we can track line numbers
          def t_newline(t):
              r'\n+'
              t.lexer.lineno += len(t.value)
              # Compute column.
              #     input is the input text string
              #     token is a token instance
          def find_column(input, token):
              line_start = input.rfind('\n', 0, token.lexpos) + 1
              return (token.lexpos - line_start) + 1
    6. 字符字面量 -> literals
              literals = [ '+','-','*','/' ]
        literals = "+-*/"
        literals = [ '{', '}' ]
        def t_lbrace(t):
            r'\{'
            t.type = '{'      # Set token type to the expected literal
            return t

        def t_rbrace(t):
            r'\}'
            t.type = '}'      # Set token type to the expected literal
            return t
    7. 错误处理
              # Error handling rule
        def t_error(t):
            print("Illegal character '%s'" % t.value[0])
            t.lexer.skip(1)
    8. EOF处理
              # EOF handling rule
        def t_eof(t):
            # Get more input (Example)
            more = raw_input('... ')
            if more:
                self.lexer.input(more)
                return self.lexer.token()
            return None
  3. 构建运行
    1. 构建lexer lexer = lex.lex()
    2. 输入要解析的字符串 lexer.input(data)
    3. 执行解析 lexer.token()

11.1.3 例子

      # ------------------------------------------------------------
      # calclex.py
      #
      # tokenizer for a simple expression evaluator for
      # numbers and +,-,*,/
      # ------------------------------------------------------------
      import ply.lex as lex

      # List of token names.   This is always required
      tokens = (
          'NUMBER',
          'PLUS',
          'MINUS',
          'TIMES',
          'DIVIDE',
          'LPAREN',
          'RPAREN',
      )

      # Regular expression rules for simple tokens
      t_PLUS    = r'\+'
      t_MINUS   = r'-'
      t_TIMES   = r'\*'
      t_DIVIDE  = r'/'
      t_LPAREN  = r'\('
      t_RPAREN  = r'\)'

      # A regular expression rule with some action code
      def t_NUMBER(t):
          r'\d+'
          t.value = int(t.value)    
          return t

      # Define a rule so we can track line numbers
      def t_newline(t):
          r'\n+'
          t.lexer.lineno += len(t.value)

      # A string containing ignored characters (spaces and tabs)
      t_ignore  = ' \t'

      # Error handling rule
      def t_error(t):
          print("Illegal character '%s'" % t.value[0])
          t.lexer.skip(1)

      # Build the lexer
      lexer = lex.lex()
      # Test it out
      data = '''
      3 + 4 * 10
        + -20 *2
      '''

      # Give the lexer some input
      lexer.input(data)

      # Tokenize
      while True:
          tok = lexer.token()
          if not tok: 
              break      # No more input
          print(tok)

11.2 yacc 语法解析器

11.2.1 用法说明

  1. 每个规则都是个函数
  2. 参数是按照顺序写的,因此用不到的数字就要跳过
      def p_expression_plus(p):
        'expression : expression PLUS term'

11.2.2 编写参考

  1. 一个典型的规则
           def p_expression_plus(p):
         'expression : expression PLUS term'
         p[0] = p[1] + p[3]
  2. 组合相似规则的方法
           def p_expression(p):
         '''expression : expression PLUS term
                       | expression MINUS term'''
         if p[2] == '+':
           p[0] = p[1] + p[3]
         elif p[2] == '-':
           p[0] = p[1] - p[3]

    也能处理不同的顺序 

           def p_expressions(p):
         '''expression : expression MINUS expression
                       | MINUS expression'''
         if (len(p) == 4):
           p[0] = p[1] - p[3]
         elif (len(p) == 3):
           p[0] = -p[2]
  3. 单个字符的简便方法

    当然这些字符都得在 lex 的 literals 里声明过 

           def p_binary_operators(p):

                       | expression '-' term
            term       : term '*' factor

         if p[2] == '+':

         elif p[2] == '-':
           p[0] = p[1] - p[3]

           p[0] = p[1] * p[3]
         elif p[2] == '/':
           p[0] = p[1] / p[3]
  4. 空捕获
           def p_empty(p):

         pass
       def p_optitem(p):
         'optitem : item'
         '        | empty'
         ...
  5. 歧义语法的处理-优先级和结合律

    需要声明 

           precedence = (
           ('left', 'PLUS', 'MINUS'),
           ('left', 'TIMES', 'DIVIDE'),
       )

    其含义为 

           PLUS      : level = 1,  assoc = 'left'
       MINUS     : level = 1,  assoc = 'left'
       TIMES     : level = 2,  assoc = 'left'
       DIVIDE    : level = 2,  assoc = 'left'

    还能在某个规则中指定特殊优先级和结合律 例如为了解析 "3 + 4 * -5" 先取负的规则 定义优先级,前面加 U 表示一元运算符 并用 %prec 重载运算符结合律

     def t_NUMBER(t):
         r'\d+'

             t.value = int(t.value)
         except ValueError:
             print("Integer value too large %d", t.value)
             t.value = 0
         return t

     # Ignored characters
     t_ignore = " \t"

     def t_newline(t):
         r'\n+'
